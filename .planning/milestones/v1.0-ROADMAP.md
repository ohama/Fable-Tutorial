# Milestone v1.0: MVP (F# 개발자를 위한 Fable 튜토리얼)

**Status:** ✅ SHIPPED 2026-06-23
**Phases:** 1-5
**Total Plans:** 18

## Overview

F#을 아는 개발자가 Fable로 실제 동작하는 웹 앱을 만들 수 있도록 돕는 한국어 예제 중심 mdBook 튜토리얼. 책 뼈대와 CI/배포 파이프라인을 먼저 완성한 뒤, 기초 축(Ch.1-3) → JS Interop 축(Ch.4-7) → Elmish/UI 축(Ch.8-10) → 생태계/실전 축(Ch.11-13) 순으로 13개 챕터를 작성. 각 챕터는 독립 실행 가능한 Fable 예제와 `{{#include}}` 앵커로 산문과 코드가 항상 동기화됨. 라이브: https://ohama.github.io/Fable-Tutorial/

## Milestone Summary

**Key Decisions:**
- 산출물 형식 mdBook 사이트 (GitHub Pages 배포 용이) — ✓ Good
- 챕터별 독립 예제 방식 (원하는 장만 골라 읽기) — ✓ Good
- 개념 설명 → 예제 코드 순서 (이해 후 확인 학습 흐름) — ✓ Good
- 코드는 {{#include}} 앵커로만 인용 (silent rot 방지) — ✓ Good (49개 앵커 전부 해석)
- `dotnet fable watch --verbose` 필수 (Fable #3631 deadlock 방지) — ✓ Good
- F# 기초는 다루지 않음 (독자 F# 숙련자 전제) — ✓ Good

**검증으로 확정된 Fable 5.3.0 실측 지식 (튜토리얼 차별점):**
- `Option.None` → `undefined` (null 아님)
- `U2<int,float>` erased union → `typeof` 충돌로 한 분기 dead code (컴파일 에러 없음)
- int64 → BigInt 리터럴, decimal → Decimal 클래스, record → ES6 Record 클래스(POJO 아님)
- Glutinum·ts2fable 모두 CommonJS `export =`를 잘못 생성 → 수기 `[<ImportDefault>]`이 정답
- Fable.Promise 3.2.0 `promise {}` CE는 Fable 5.3.0 호환 (단 `open Promise` 금지)
- Feliz.Router 4.0.0 ↔ Feliz 3.3.3 비호환 → 수제 hash 라우팅으로 대체
- `--noReflection` ↔ `Decode.Auto` 충돌 → 수동 Thoth 디코더 채택
- Fable.Mocha 2.17.0 ↔ Fable 5.3.0 호환 확인 (mocha 9.2.0 핀, 출력 경로 dist/src/*.js)

**Issues Resolved:**
