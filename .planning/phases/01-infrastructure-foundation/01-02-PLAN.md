<task type="auto">
  <name>Task 1: Build custom highlight.js with fsharp and place in theme/</name>
  <files>theme/highlight.js</files>
  <action>
Build a highlight.js bundle that includes the `fsharp` language, per RESEARCH Pattern 3 Option A:

```bash
git clone https://github.com/highlightjs/highlight.js.git --depth 1 /tmp/hljs
cd /tmp/hljs
npm install
node tools/build.js fsharp
# output: /tmp/hljs/build/highlight.min.js
```

Copy the produced bundle to the repo at `theme/highlight.js` (create the `theme/` directory at repo root, same level as book.toml — RESEARCH structure):
```bash
mkdir -p <repo-root>/theme
cp /tmp/hljs/build/highlight.min.js <repo-root>/theme/highlight.js
```

If `node tools/build.js fsharp` is unavailable in the cloned version, fall back to RESEARCH Pattern 3 Option B: download a custom build from https://highlightjs.org/download selecting "fsharp" from Functional languages, then rename the bundle to `highlight.js` and place it at `theme/highlight.js`. Either way, the final file MUST register the fsharp language.

Do NOT add any book.toml configuration — mdBook detects `theme/highlight.js` automatically.
  </action>
  <verify>
`test -f theme/highlight.js`. The file is non-trivial (`wc -c theme/highlight.js` is large, i.e. a real minified bundle not an empty file). The fsharp language is present: `grep -iq 'fsharp\|f#' theme/highlight.js`.
  </verify>
  <done>theme/highlight.js exists at repo root, is a real highlight.js bundle, and contains the fsharp language definition.</done>
